# Task Startup Checklist

Use this checklist before and after every formal development task.

## Before Implementation
1. Read `AGENTS.md`, `PLANNING.md`, and `DEVLOG.md`.
2. Read `notes/research_context.md`.
3. Reopen the source materials most relevant to the task.
4. Update `PLANNING.md` with the current goal, scope, files, and validation plan.
5. State how the task supports the Phase I benchmark-first roadmap.

## After Implementation
1. Run the smallest meaningful validation for the task.
2. Update `DEVLOG.md` with changes, evidence, and next steps.
3. Confirm whether any config, docs, or templates need follow-up updates.
4. Keep the next task small, explicit, and research-linked.
